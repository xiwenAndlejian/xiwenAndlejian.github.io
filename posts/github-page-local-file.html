Github Page 使用静态文件

12/09/2018 #jekyll #github page

本地运行 jekyll 和 Github Page 显示效果不一致 发现是由于 Github Page 不支持第三方插件导致的 而想要使用插件效果，只能使用 jekyll 编译生成的静态文件

jekyll使用静态文件

参考

我的过程和参考中的内容稍有不同，主要是希望把

  1. 将当前文件夹重命名为compiled_site

  2. 在当前文件夹外创建一个文件夹jekyll_site

  3. 将compiled_site的内容移动到jekyll_site内

  4. 再将compiled_site移动到jekyll_site内

  5. 进入compiled_site中，使用git rm -r删除所有文件

  6. 在compiled_site中创建.nojeyll（空文件即可）

  7. 在jekyll_site中创建Rakefile 

    desc "Build Jekyll site and copy files"
task :build do
  system "jekyll build"
  system "cp .nojeyll _site"
end

  8. 运行rake build

  9. 再到_site中执行git add --all :/

  10. 提交git即可
